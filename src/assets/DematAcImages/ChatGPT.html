The issue where the icons are coming in a new line instead of in a flex row direction is likely due to the p tags around the icons. In HTML, p elements are block-level elements by default, which means they take up the full width available and start on a new line. This default behavior can override the flexbox layout.

To fix this, you can change the p tags to inline elements such as span or add a class to make the p elements inline. Here's how you can do both approaches:

Approach 1: Use span Tags

Replace the p tags with span tags:

jsx
<div className="sm:hidden mobile:hidden md:block w-[80%] flex justify-end mt-4">
  <span onClick={handlePrev}>
    <PiCaretLeftBold size={25} className="hover:text-purple"/>
  </span>
  <span onClick={handleNext}>
    <PiCaretRightBold size={25} className="hover:text-purple"/>
  </span>
</div>

Approach 2: Make p Tags Inline

Add a class to the p tags to make them inline:

jsx
<div className="sm:hidden mobile:hidden md:block w-[80%] flex justify-end mt-4">
  <p className="inline" onClick={handlePrev}>
    <PiCaretLeftBold size={25} className="hover:text-purple"/>
  </p>
  <p className="inline" onClick={handleNext}>
    <PiCaretRightBold size={25} className="hover:text-purple"/>
  </p>
</div>

Then add the CSS for the .inline class:

css
.inline {
  display: inline;
}

Tailwind CSS Approach

If you are using Tailwind CSS, you can directly apply the inline class to the p tags:

jsx
<div className="sm:hidden mobile:hidden md:block w-[80%] flex justify-end mt-4">
  <p className="inline" onClick={handlePrev}>
    <PiCaretLeftBold size={25} className="hover:text-purple"/>
  </p>
  <p className="inline" onClick={handleNext}>
    <PiCaretRightBold size={25} className="hover:text-purple"/>
  </p>
</div>

By ensuring the p tags are inline elements, they will align correctly within the flex container.

To create a dropdown for each footer section in the mobile view, you can use the useState hook to manage the visibility of the subtitles for each footer section. Here's how you can achieve this:

  1. Add State Management: Use a state to track which footer section is open.
  2. Conditional Rendering: Conditionally render the subtitles based on the state.
  3. Toggle Function: Create a function to toggle the visibility of the subtitles.

Implementation

Here's how you can implement this functionality:

jsx
import { useState } from "react";
import { FooterData } from "../data/data";

const Footer = () => {
  const [openSections, setOpenSections] = useState({});

  const toggleSection = (id) => {
    setOpenSections((prev) => ({
      ...prev,
      [id]: !prev[id],
    }));
  };

  return (
    <div className="ml-[4%] flex w-[70%] flex-col md:flex-row">
      <div className="w-full flex flex-col md:flex-row justify-between">
        {FooterData.map((footerHeadersData) => (
          <div className="mb-4 md:mb-0" key={footerHeadersData.id}>
            <h1
              className="mb-4 font-bold text-lg text-[#30303C] cursor-pointer"
              onClick={() => toggleSection(footerHeadersData.id)}
            >
              {footerHeadersData.title}
            </h1>
            {openSections[footerHeadersData.id] && (
              <div>
                {footerHeadersData.subtitle.map((subtitle, index) => (
                  <div key={index} className="py-1.5">
                    <p className="text-base text-[#30303C]">
                      {subtitle.heading}
                    </p>
                  </div>
                ))}
              </div>
            )}
          </div>
        ))}
      </div>
    </div>
  );
};

export default Footer;

Explanation

  1. State Management:

    • openSections is an object where each key corresponds to a footer section's ID and the value is a boolean indicating whether the section is open.
    • setOpenSections updates this state.

  2. Toggle Function:

    • toggleSection toggles the boolean value for a specific section ID in the openSections object.

  3. Conditional Rendering:

    • {openSections[footerHeadersData.id] && (...)} conditionally renders the subtitles if the corresponding section is open.
    • Clicking on the section title (h1 tag) triggers toggleSection to open/close the section.

import React, { useState, useEffect } from "react";
import { GrFormNext, GrFormPrevious } from "react-icons/gr";

function CustomCarousel({ children }) {
  const [activeIndex, setActiveIndex] = useState(0);
  const [slideDone, setSlideDone] = useState(true);
  const [timeID, setTimeID] = useState(null);
  const [isMobile, setIsMobile] = useState(window.innerWidth <= 768);

  useEffect(() => {
    const handleResize = () => {
      setIsMobile(window.innerWidth <= 768);
    };

    window.addEventListener("resize", handleResize);
    return () => window.removeEventListener("resize", handleResize);
  }, []);

  useEffect(() => {
    if (slideDone) {
      setSlideDone(false);
      setTimeID(
        setTimeout(() => {
          slideNext();
          setSlideDone(true);
        }, 5000)
      );
    }
  }, [slideDone]);

  const slideNext = () => {
    setActiveIndex((val) => {
      if (val >= children.length - 1) {
        return 0;
      } else {
        return val + 1;
      }
    });
  };

  const slidePrev = () => {
    setActiveIndex((val) => {
      if (val <= 0) {
        return children.length - 1;
      } else {
        return val - 1;
      }
    });
  };

  const AutoPlayStop = () => {
    if (timeID > 0) {
      clearTimeout(timeID);
      setSlideDone(false);
    }
  };

  const AutoPlayStart = () => {
    if (!slideDone) {
      setSlideDone(true);
    }
  };
